반복적, 점진적 개발

### 종류 
- [[Spiral model]]
- Unified Process (RUP)


### 장점
- 위험 감소
- 수정 유연성
- 효율적 피드백 (big bang approach나 waterfall 모델과 달리)
- 높은 퀄리티

### 단점
- 아키텍쳐/ 디자인이 최적이 아닐 수 있음.
- rework와 total cost가 높을 수 있음.

### 응용 가능성
- response time이 중요할 때
- 긴 프로젝트의 위험이 허용되지 않을 때
- 요구 사항을 알지 못 할 때
